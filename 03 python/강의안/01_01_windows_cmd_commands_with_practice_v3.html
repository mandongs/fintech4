Windows CMD 강의안 — 명령어 + 실습 코드

“명령어 설명”만 나열하지 않고, 각 명령어마다 바로 실행 가능한 실습 코드를 함께 제공하는 강의안이다. 실습은 동일한 작업 폴더에서 진행하는 것이 기준이다.

실습 준비(필수) 관리자 권한 vs 일반 모드 경로/탐색 폴더/파일 관리 복사/이동 리다이렉션/로그 시스템/환경변수 네트워크 미니 프로젝트 실습

실습 준비(필수)

실습 폴더를 고정하는 이유

CMD는 “현재 작업 폴더” 기준으로 파일/폴더가 생성된다. 특히 관리자 권한으로 실행하면 시작 위치가 C:\Windows\System32인 경우가 많아 실습 결과물이 엉뚱한 곳에 생성될 수 있다.

실습 준비 코드(그대로 복사해서 실행) 
cd
cd C:\
mkdir fintech_cmd_lab
cd fintech_cmd_lab
cd
체크 마지막 cd 출력이 C:\fintech_cmd_lab라면 준비가 완료된 상태이다.

입력 편의 기능(탭 자동완성, 방향키 히스토리)

TAB 키 — 경로/파일명 자동완성

CMD에서는 파일명·폴더명·경로를 입력할 때 TAB 키를 누르면 자동완성이 된다. 같은 글자로 시작하는 항목이 여러 개면 TAB을 반복해서 누를 때마다 다음 후보로 이동한다. Shift + TAB을 누르면 반대로(이전 후보) 이동한다.

실습 코드(탭 자동완성 체험) 
cd C:\\fintech_cmd_lab
mkdir tab_test
cd tab_test
mkdir alpha beta beta_log beta_tmp

cd b    (여기서 TAB을 눌러서 beta로 자동완성)
cd ..
팁 긴 경로를 직접 타이핑하지 말고, 폴더명의 앞부분만 입력한 다음 TAB을 쓰는 것이 훨씬 빠르고 실수가 줄어든다.

방향키 ↑ ↓ — 이전/다음 명령 불러오기

CMD에서 위 방향키(↑)를 누르면 이전에 실행했던 명령을 한 줄씩 다시 불러올 수 있다. 아래 방향키(↓)는 다시 앞으로(최근 명령 쪽으로) 이동한다. 같은 폴더에서 반복 작업할 때 가장 많이 쓰는 기능이다.

실습 코드(히스토리 체험) 
cd C:\\fintech_cmd_lab
dir
tree /f
echo Hello > hist.txt
type hist.txt

(이제 ↑ 키를 여러 번 눌러서 방금 실행한 명령들이 다시 나타나는지 확인한다)
(원하는 명령이 나타나면 Enter로 다시 실행한다)
추천 습관 실습 중 오타가 났다면 “다시 타이핑”하지 말고 ↑로 불러온 뒤 일부만 수정하는 것이 효율적이다.

추가 팁: doskey /history — 명령 기록 목록 보기

CMD에는 세션(현재 창) 기준으로 명령 기록이 남는다. 아래 명령으로 지금까지 친 명령 목록을 볼 수 있다.

실습 코드 
doskey /history
참고 CMD 창을 닫으면 일반적으로 히스토리는 초기화된다(세션 단위이다).

관리자 권한 vs 일반 모드

개념

  • 일반 모드는 현재 사용자 권한으로만 명령을 실행하는 모드이다.
  • 관리자 권한 모드는 시스템 관리자 권한으로 명령을 실행하는 모드이다.
  • 관리자 모드는 시스템 폴더/설정 관련 작업을 수행할 수 있지만 실습 결과물이 시스템 폴더에 생성되는 실수가 잦다.

실습: “시작 위치가 다르다” 확인하기

일반 모드 CMD와 관리자 권한 CMD를 각각 열고 아래 코드를 실행해 출력이 어떻게 다른지 비교한다.

실습 코드 
cd
dir
정리 모드와 상관 없이 실습은 항상 cd C:\fintech_cmd_lab로 위치를 고정하고 시작하는 것이 안전하다.

경로 구분자 \\ 와 / 의 차이

Windows CMD의 기본 경로 구분자: \\

Windows CMD에서 경로를 구분하는 기본 문자는 백슬래시(\\)이다. 폴더 계층을 표현할 때 표준적으로 사용하며, 대부분의 CMD 명령어 예제는 \\를 기준으로 작성된다.

실습 코드(정상 동작 예) 
cd C:\\fintech_cmd_lab
mkdir path_test\\sub1
dir path_test
cd path_test\\sub1
cd ..
정리 CMD 실습에서는 경로 입력 시 기본적으로 \\를 사용하는 것이 가장 안전하다.

/ (슬래시)가 헷갈리는 이유

CMD에서 / 문자는 보통 옵션(option) 구분자로 사용된다. 따라서 dir /w, ipconfig /all처럼 명령어 뒤에 붙어 “옵션”의 의미로 해석되는 경우가 많다.

실습 코드(옵션으로 쓰이는 /) 
dir /w
tree /f
ipconfig /all
중요 CMD는 기본적으로 /를 “경로”가 아니라 “옵션”으로 인식한다.

왜 어떤 명령에서는 / 경로도 되는 것처럼 보일까?

일부 Windows 프로그램이나 최신 명령어는 내부적으로 /를 \\로 자동 변환해 처리하기도 한다. 이 때문에 cd C:/Windows 같은 입력이 동작하는 경우가 있다.

실습 코드(혼용 가능 사례) 
cd C:/Windows
cd C:\\Windows
주의 모든 명령에서 항상 허용되는 것은 아니다. 초보 단계에서는 \\만 사용하도록 습관을 들이는 것이 좋다.

Linux / macOS 와의 차이

Linux와 macOS의 터미널에서는 경로 구분자로 /만 사용한다. Windows CMD와 가장 헷갈리는 지점 중 하나이다.

비교 예시 
Windows CMD : C:\\Users\\user\\Documents
Linux/macOS : /home/user/Documents
기억 포인트
  • Windows CMD 경로 → \\
  • CMD 옵션 → /
  • Linux/macOS 경로 → /

경로/탐색

cd — 현재 경로 확인/이동

cd는 현재 작업 폴더를 보여주거나, 원하는 폴더로 이동하는 명령어이다.

실습 코드 
cd
cd C:\
cd fintech_cmd_lab
cd
cd .. 
cd
포인트 cd ..는 “상위 폴더로 이동”이다. 실습 중 경로가 꼬이면 cd C:\fintech_cmd_lab로 복귀하면 된다.

dir — 목록 보기

dir는 현재 폴더(또는 지정한 폴더)의 파일/폴더 목록을 출력하는 명령어이다.

실습 코드 
cd C:\fintech_cmd_lab
dir
dir /w
dir /p
실습 코드(특정 폴더를 지정해서 보기) 
mkdir sub1
mkdir sub2
dir sub1
dir sub2

tree — 폴더 구조 보기

tree는 폴더 구조를 트리 형태로 출력하는 명령어이다. /f 옵션은 파일까지 포함한다.

실습 코드 
cd C:\fintech_cmd_lab
tree
tree /f
tree C:\fintech_cmd_lab
tree C:\fintech_cmd_lab /f

폴더/파일 관리

mkdir — 폴더 만들기

mkdir는 폴더를 만드는 명령어이다. 여러 폴더를 한 번에 만들 수도 있다.

실습 코드 
cd C:\fintech_cmd_lab
mkdir test
mkdir test\subA test\subB
tree test

rmdir / rd — 폴더 삭제

rmdir는 폴더를 삭제하는 명령어이다. rd는 rmdir의 축약이다.

실습 코드(빈 폴더 삭제) 
cd C:\fintech_cmd_lab
mkdir emptyFolder
dir
rmdir emptyFolder
dir
실습 코드(하위 포함 전체 삭제) 
cd C:\fintech_cmd_lab
mkdir deltest\sub1
echo hello > deltest\sub1\a.txt
tree deltest /f
rmdir /s /q deltest
dir
주의 rmdir /s /q는 “되돌리기”가 어렵다. 실습 폴더(C:\fintech_cmd_lab)에서만 실행하는 것이 안전하다.

echo / type / copy con — 파일 만들기와 내용 확인

파일을 빠르게 만들 때는 echo + 리다이렉션을 많이 사용한다. 파일 내용을 확인할 때는 type을 사용한다. 직접 입력으로 파일을 만들 때는 copy con을 사용할 수 있다.

실습 코드(echo + > / >>) 
cd C:\fintech_cmd_lab
echo Hello > a.txt
echo World >> a.txt
type a.txt
실습 코드(copy con) 
cd C:\fintech_cmd_lab
copy con b.txt
(여기에 내용을 입력한다)
Ctrl+Z 누르고 Enter
팁 수업 실습에서는 echo 방식이 더 빠르다. copy con은 “직접 입력” 실습용으로 좋다.

del / rename — 파일 삭제/이름 변경

실습 코드 
cd C:\fintech_cmd_lab
echo data > old.txt
dir
rename old.txt new.txt
dir
del new.txt
dir

복사/이동

copy — 파일 복사

실습 코드 
cd C:\fintech_cmd_lab
echo copytest > c1.txt
copy c1.txt c2.txt
type c2.txt

move — 파일/폴더 이동

실습 코드(폴더로 이동) 
cd C:\fintech_cmd_lab
mkdir moveTo
echo hello > m.txt
move m.txt moveTo\
dir moveTo
실습 코드(상위 폴더로 이동: ..\) 
cd C:\fintech_cmd_lab\moveTo
echo hi > up.txt
move up.txt ..\
cd ..
dir

xcopy / robocopy — 폴더 복사

xcopy는 전통적인 폴더 복사 명령어이고, robocopy는 대량 복사에 강하다.

실습 코드(xcopy) 
cd C:\fintech_cmd_lab
mkdir src\sub
echo file1 > src\a.txt
echo file2 > src\sub\b.txt
xcopy src backup /e
tree backup /f
실습 코드(robocopy) 
cd C:\fintech_cmd_lab
robocopy src backup2 /e
tree backup2 /f

리다이렉션/로그

> 와 >> — 결과를 파일로 저장

>는 덮어쓰기 저장이다. >>는 기존 파일에 추가 저장이다.

실습 코드(덮어쓰기 저장) 
cd C:\fintech_cmd_lab
dir > files.txt
type files.txt
실습 코드(추가 저장) 
cd C:\fintech_cmd_lab
echo ---- >> files.txt
dir >> files.txt
type files.txt

type nul > — 빈 파일 만들기

실습 코드 
cd C:\fintech_cmd_lab
type nul > empty.txt
dir
type empty.txt

시스템/환경변수

hostname / ver / systeminfo

실습 코드 
hostname
ver
systeminfo

환경변수: %username% / %computername%

환경변수는 %변수명% 형태로 출력한다.

실습 코드 
echo %username%
echo %computername%

date / time

실습 코드 
date
time
주의 일부 환경에서는 날짜/시간 변경 시 권한이 필요할 수 있다. 실습은 “확인”까지만 진행하는 것이 안전하다.

네트워크

ipconfig / ipconfig /all

실습 코드 
ipconfig
ipconfig /all

ping

실습 코드 
ping www.google.com
실습 코드(결과를 파일로 저장) 
ping www.google.com > ping.txt
type ping.txt

미니 프로젝트 실습

목표

아래 코드를 순서대로 실행해 “폴더 만들기 → 파일 생성 → 구조 출력 → 로그 저장”을 한 번에 수행한다. 실습문제 50의 핵심 흐름을 그대로 담은 미니 프로젝트이다.

미니 프로젝트 실행 코드 
cd C:\
mkdir fintech_cmd_lab
cd fintech_cmd_lab

mkdir project\data project\logs

echo Hello > project\data\a.txt
echo World >> project\data\a.txt
type project\data\a.txt

tree project /f > project\logs\tree.txt
dir project\data > project\logs\files.txt
ipconfig > project\logs\ip.txt
ping www.google.com > project\logs\ping.txt

dir project\logs
type project\logs\tree.txt
확장 과제 마지막에 dir >> project\logs\files.txt를 여러 번 실행해 누적 저장을 확인해도 된다.
출처: 업로드된 CMD 강의안 + 실습문제(50) 커버 범위를 기준으로 실습 중심으로 재구성한 문서이다.
